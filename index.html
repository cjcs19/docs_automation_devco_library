Automation Devco Library Documentation

1. Overview

Thank you so much for read and use our library.

  • Version: 2.0.2
  • Author: Devco
  • Created: 23 July, 2019
  • Update: 30 July, 2021

If you have any questions that are beyond the scope of this documentation, Please feel free to email via Contact section.

Introduction

The purpose of this library is to improve your automation script if you are using SerenityBDD framework and Screenplay as a pattern. You can read more about this technologies here:

  • SerenityBDD
  • Screenplay Pattern
  • Java

Motivation

¿Who has not dreamed of having superpowers?

We know that we don´t have the possibility to give you extra force, ability to fly or get invisible but something that We can do is give you superpowers to your automation code.
Yeah, that's right. How many times have you noticed that every time you create a web or mobile automation project, you end up developing new functions for your actors? Serenity Screenplay is a powerful tool that allows you to develop highly readable and reusable tests. But sometimes, we need a little more abilities or performable actions to make our suite much more adjusted to the user's journey through the application. At Devco we are superpower makers, so we've developed a library that is well used with Screenplay, will make any user invincible on this journey. And you can also use it in single/cross-platform test suites. Awesome huh? So enough business talk. Let's get our hands dirty.

2. Getting Started

Setup

This library is primarily used as a Java dependency. We typically use a build tool (such as Maven or Gradle) to resolve the Automation Devco Library dependency.
If you are using Gradle it can be done as follows 

                
                    dependencies {
                        implementation group: 'co.com.devco.automation', name: 'automationDevcoLibrary', version: '2.0.2'
                    }

In case that you are using Maven: 

                
                  <dependency>
                    <groupId>co.com.devco.automation </groupId>
                    <artifactId>automationDevcoLibrary </artifactId>
                    <version>2.0.2 </version>
                  </dependency>
